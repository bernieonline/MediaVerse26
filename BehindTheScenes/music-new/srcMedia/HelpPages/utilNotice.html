Welcome to the MediaManager Utilities Section

This is where you can manage your collections. Remove - Delete - Move - Transfer

Here's how it works:

  • The Menu provides these facilities Remove - Delete - Move
  • The Left Pane shows a list of collections
  • The actions taken when you click on a row largely determine what happens when you click on a collection row
  • The exception is the Transfer option. This is achieved by Right Click on a row
  • This provides a choice of moving a collection from the Mediafiledetail to the Masterfiledetail, or the reverse, it does not move actual media files only the records
  • The Remove option will delete all of the rows in the database related to the chosen collection, again the media files will be unaffected only the collection records are removed
  • They can be updated by recreating the collection. This is aften needed because any changes made to the location of media files made after creating a collection will require you to delete the old collection records and recreate the modified set
  • The Delete and Move options are identical in the way that they operate
  • These are manual transactions. The objective is to Delete or Move the underlying DATA so care must be taken as the process can't be reversed

When you choose to Delete or Move a collection, a File Explorer Window opens at the location of the collection, you must make the changes manually

What's this for

Once you have established a set of Master Files, the remaining collections should probably be removed otherwise you are wasting storage space

Delete those collections no longer needed in order to save space or time in maintaining synchronicity with the master sets

The alternative is to Move the files to an archive folder such as a NAS storage system.

And as we said earlier, if you move individual records, perhaps into a master set to update it, you will need to remove the old master set records and rebuild the set in order to incorporate the new data.

It's the same with any media files that you delete, you will need to remove the records from the database by Removing a collection and re creating it to reflect the changes
